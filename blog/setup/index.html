    Setup a blog with Material for MkDocs¶

    Install Material For MkDocs¶

    Install Python¶

    Download Python from https://www.python.org/downloads/ and install it.

    Make sure to add Python installation folder into environmental variable PATH

    Prepare environment¶

    1. Create a new folder and initialize a virtual environment 

      python -m venv .venv

    2. Activate the virtual environment

      .venv\Scripts\activate.bat

    3. Update pip

      python -m pip install -U pip

    4. Install Material for MkDocs

      pip install mkdocs-material
      Installed packages

      mkdocs-material will automatically install its dependencies including

      Jinja2
livereload
lunr
Markdown
MarkupSafe
mkdocs
mkdocs-material
mkdocs-material-extensions
Pygments
pymdown-extensions
PyYAML
tornado

    Initialize your site¶

    1. Create a new MkDocs site

      mkdocs new .

      this will create new files in the following structure

      .
├─ docs/
│  └─ index.md
└─ mkdocs.yml

    2. Enable Material for MkDocs theme in mkdocs.yml setting file

      theme:
    name: material

    3. Build and serve on localhost:8000

      mkdocs serve

      Point your browser to and see your page.

palette:

    accent: indigo

  - stylesheets/extra.css

    then define colors in extra.css

    :root {
    --md-primary-fg-color:        #EE0F0F;
    --md-primary-fg-color--light: #ECB7B7;
    --md-primary-fg-color--dark:  #90030C;
}

    Set in mkdocs.yml:

    theme:
  font:
    text: Roboto
    code: Roboto Mono

    Critic¶

    Set in mkdocs.yml:

    markdown_extensions:
  - pymdownx.critic:
      mode: view

    Usage:

    Text can be {--deleted--} and replacement text {++added++}. This can also be combined into {~~one~>a single~~} operation. {==Highlighting==} is also possible {>>and comments can be added inline<<}.

    {==

    Formatting can also be applied to blocks, by putting the opening and closing tags on separate lines and adding new lines between the tags and the content.

    Text can be deleted and replacement text added. This can also be combined into onea single operation. Highlighting is also possible and comments can be added inline.

    EscapeAll¶

    Set in mkdocs.yml:

    markdown_extensions:
  - pymdownx.escapeall:

    There are two special escapes among all of these escapes though: escaping "space" characters and escaping "newline" characters. If nbsp is enabled, an escaped space will be converted into a non-breaking space: &nbsp;. If hardbreak is enabled, an escaped newline will be converted to a hard break <br>. The advantage of hardbreak is that you can visually see the hard break opposed to Markdown's default method of two spaces at the end of a line.

    Usage:

    This is a line with some spaces \ \ \ \ \  in this line\
This is a line with some spaces            in this line

A new line is here\
A new line is here
The last line!

    This is a line with some spaces       in this line
    This is a line with some spaces in this line

    A new line is here
    A new line is here The last line!

    Magic Link¶

    MagicLink supports auto-linking HTTP, FTP and email links. You can specify these links in raw text and they should get auto-linked. There are some limitations placed on MagicLink to keep it from aggressively auto-linking text that is not part of links. If you have a link that cannot be detected, you can always use the old style angle bracketed link format: https://www.link.com

    Set in mkdocs.yml:

    markdown_extensions:
  - pymdownx.magiclink:

    KbdExtention¶

    The KBD Extension is an inline processor for adding markdown syntax for inline <kbd> tags. Text wrapped in double brackets [[ button ]], double braces {{ menu }}, or double parenthesis (( action )) will be wrapped with an HTML <kbd> tag.

    Install

    pip install kbdextension

    Set in mkdocs.yml:

    markdown_extensions:
  - kbdextension:
      brackets_css: button # [[button]]
      enable_parens: true
      parens_css: menu # ((menu))
      enable_braces: true
      braces_css: action # {{action}} or {{ "{{action}}" }}when using marcro
    KBD Extention CSS
    .md-typeset kbd {
    padding: 0 0.5em;
    font-size: .80em;
    box-shadow: unset;
}

    color: white;
    background-color: dimgray;
}
.md-typeset kbd.menu {
    color: dodgerblue;
    border: 1px dotted  dodgerblue;
}

.md-typeset kbd.button {
    color: white;
    background-color: dodgerblue;
}

.md-typeset kbd.action {
    color: indigo;
    border-bottom: 1px dotted  dodgerblue;
}

    Usage:

    Keys: ++ctrl+alt+del++ or ++shift+a++ \
Button: [[Download]] or [[Details]] \
Menu: ((Save)) or ((Run > Run As)) \
Action: {{Double click}} or {{Long press}}

    Button: Download or Details
